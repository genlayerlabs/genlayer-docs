# Running GenLayer Simulator on GitHub Actions and exposing it via Ngrok

### Step 1: Fork the Repository
- Fork the GenLayer Simulator repository from the [official GenLayer GitHub account](https://github.com/yeagerai/genlayer-simulator/) to your personal GitHub account.

### Step 2: Register with Ngrok and configure GitHub Actions secrets
1. **Create an Ngrok account:**
   - Sign up at [ngrok.com](https://ngrok.com) and log in to access your dashboard.
2. **Retrieve Ngrok Auth Token:**
   - From the Ngrok dashboard, copy your unique authentication token.
3. **Add Auth Token to GitHub Secrets:**
   - Navigate to your forked repository on GitHub.
   - Go to **Settings > Secrets and variables > Actions**.
   - Add a new secret named `NGROK_AUTHTOKEN` and paste the copied token.
4. **Adjust Your Ngrok Domain:**
   - Replace `--domain=pipefish-promoted-chipmunk.ngrok-free.app` with your own URL in the configuration.

### Step 3: Add GitHub Actions Pipeline
- Create a `.github/workflows/simulator.yml` file in your repository with the following content and commit it:

```yaml
name: Run simulator with ngrok

on:
  push:
    branches:
      - main
  workflow_dispatch:

jobs:
  simulator-ngrok:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Copy .env file
        run: cp .env.example .env

      - name: Changing URL for rpc server to localhost
        run: sed -i "s/'jsonrpc'/'localhost'/g" .env

      - name: Remove OPENAIKEY from the .env file so it can be loaded from the environment
        run: sed -i "/^OPENAIKEY *= *'<add_your_api_key_here>'$/d" .env

      - name: Adding OPENAIKEY to the .env file
        env:
          OPENAIKEY: ${{ secrets.OPENAIKEY }}
        run: printf "\nOPENAIKEY='$OPENAIKEY'\n" >> .env

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Cache Docker layers
        uses: actions/cache@v4
        with:
          path: /tmp/.buildx-cache
          key: ${{ runner.os }}-buildx-${{ github.sha }}
          restore-keys: |
            ${{ runner.os }}-buildx-

      - name: Run Docker Compose
        run: docker compose up -d

      - name: Wait for services to be up
        run: |
          timeout=60
          counter=0
          while [[ "$counter" -lt "$timeout" ]]; do
            if curl -X POST -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","method":"ping","params":[],"id":1}' http://0.0.0.0:4000/api | grep -q "OK"; then
              echo "RPC server is up!"
              break
            else
              echo "Waiting for RPC server... ($counter/$timeout)"
              sleep 5
              counter=$((counter+1))
            fi
          done
          # Fail if the service didn't start within the timeout
          if [[ "$counter" -ge "$timeout" ]]; then
            echo "Error: Timeout while waiting for RPC server"
            exit 1
          fi

      - name: Run ngrok tunnel
        timeout-minutes: 240 # The maximum time to run this job
        run: |
          docker run --net=host -e NGROK_AUTHTOKEN=${{ secrets.NGROK_AUTHTOKEN }} ngrok/ngrok:latest http --domain=pipefish-promoted-chipmunk.ngrok-free.app 8080

      - name: Shutdown Docker Compose
        if: always()
        run: docker compose down
```

### Step 4: Access the Simulator
- Open your web browser and navigate to your Ngrok URL, e.g., `https://pipefish-promoted-chipmunk.ngrok-free.app/`.
- Click on **Settings** in the left corner and add your validators.
