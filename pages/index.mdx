## GenLayer FAQ
The GenLayer FAQ is a collection of useful questions and answers about the project. If you have a question that isn't answered here, please let us know on our [Discord](https://discord.gg/8Jm4v89VAu) .

<details style={{ backgroundColor: '#262626', padding: '10px', borderRadius: '5px', color: 'white' }}>
  <summary style={{ color: 'white' }}>
    <b>How do I set up the GenLayer environment?</b>
  </summary>
  To quickly set up the GenLayer environment, run the commands below:
  ```
  $ npm install -g genlayer
  $ genlayer init
  ```
  For more detailed setup instructions, please refer to the [Install GenLayer page](https://docs.genlayer.com/simulator/installation)
</details>

<details style={{ backgroundColor: '#262626', padding: '10px', borderRadius: '5px', color: 'white' }}>
  <summary style={{ color: 'white' }}>
    <b>Where can I find the GenLayer Simulator?</b>
  </summary>
  For you to use the simulator, you need to install GenLayer via the [CLI](http://localhost:3000/simulator/installation#installation-steps). Once the setup is complete, you can access the GenLayer Simulator at http://localhost:8080/. 
  </details>

<details style={{ backgroundColor: '#262626', padding: '10px', borderRadius: '5px', color: 'white' }}>
  <summary style={{ color: 'white' }}>
    <b>How do I deploy an Intelligent Contract?</b>
  </summary>
   - Use the CLI to start the GenLayer Simulator.
   - Write your contract in Python using the [Intelligent Contract syntax](https://docs.genlayer.com/simulator/intelligent-contract-syntax). 
   - Folllow the guide [here](https://docs.genlayer.com/simulator/usage-and-interaction) to interact with the simulator and deploy your Intelligent Contract.
</details>

<details style={{ backgroundColor: '#262626', padding: '10px', borderRadius: '5px', color: 'white' }}>
  <summary style={{ color: 'white' }}>
    <b>What is the GenLayer CLI used for?</b>
  </summary>
  The GenLayer CLI is used for setting up the GenLayer Simulator and, in the future, will support mainnet and testnet environments. 
</details>
